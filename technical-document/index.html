Getting Started with Java

Java is a high-level, class-based, object-oriented programming language that is designed to have as few dependencies as possible.

  1. Install Java Development Kit (JDK)
  2. Set up your Integrated Development Environment
  3. Start your project in your IDE
  4. Make sure you save and compile your project
  5. Run the code and debug any issues
Java Entry Point

The entry point of a Java application is the main method or function. It is where the program starts executing.

It is categorized by its (String[] args) as a parameter. This parameter "[]" is an array (collection) of strings that can be used to pass command-line arguments to the program.

public class myMainFunction {
    public static void main(String[] args) {
        /* Java main function example */
    }
}

As you can see, the main function is wrapped within a class which is part of the object-oriented structure of Java.

The class name must match the file name.

For example, if the class is named "myMainFunction", the file should be named "myMainFunction.java".

Java is case-sensitive, so "myMainFunction" and "mymainfunction" would be considered different names.

Java uses the "public" access modifier to allow the main method to be accessible from outside the class.

The "static" keyword means that the main method can be called without creating an instance of the class.

The "void" keyword indicates that the main method does not return any value.

The "String[] args" parameter allows the program to accept command-line arguments when it is run.

Printing to the console

In order to print to the console, you can use the System.out.println() method.

This method prints the specified message to the console.

public class myMainFunction {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

In this example, the message "Hello, World!" will be printed to the console when you run the program.

Object-Oriented Programming

Java is known as an object-oriented programming language.

This means that it is easy to represent real-world objects and concepts in code by using classes and encapsulation.

Encapsulation in Java refers to bundling data (attributes/variables) and the methods that operate on that data into a single unit (class).

An example of this might be a Student class to represent a student.

public class Student {
    // Attributes
    private String name;
    private int age;

    // Constructor
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Getter method
    public String getName() {
        return name;
    }

    // Setter method
    public void setName(String name) {
        this.name = name;
    }
}

We use this class by doing the following:

Student student1 = new Student("Johnny Bravo", 20);
String johnnyName = student1.getName();
student1.setName("Jimmy");
String jimmyName = student1.getName();
